About Me

I’m an agile software engineer. I’m a strong advocate and practitioner of XP and swear by the practices of Test Driven Development, Pairing and CI/CD. I’m passionate about building stuff, solving problems, and learning new skills while mastering the skills I have.

Effective and fast CI/CD is critical for a project of any size. I have been thinking a lot about how to make feedback loops as short as possible and incorporating that into more frequent and guided iterations.

I have been diving deep on Test Driven Development and coupling this with studying coaching. I want to become an expert teacher and be able to enable others to practice TDD as effectively as my colleagues enabled me.

I am very interested in all pieces that go into building great software. I have been working to build the ability to think in terms of Lean Product Development using build-measure-learn cycles . Building tangible products for real users is what excites me the most about software development. I’ve been working on ui-heavy personal projects in my own time to hone my frontend development skills, along with actively practicing backlog management, CI/CD, and design.

Most of my professional development experience has been with Java using the Spring Framework. I’ve been working on event drive APIs using a microservie approach hosted in Pivotal Cloud Foundry. I have spent a good about of time working with React and AngularJS (1). In school I worked mostly in Python or C++.
